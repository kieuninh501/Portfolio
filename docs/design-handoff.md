# Portfolio Design Handoff

Use this checklist before sending the Figma file so the implementation can stay clean.

## Required Figma Inputs

- Desktop frame: 1440 px wide.
- Mobile frame: 390 px wide.
- Color styles or variables for background, surface, text, muted text, border, accent, and focus state.
- Text styles for display, section heading, body, small label, and button.
- Components for button, card, badge, navbar, project card, and section heading.
- Auto layout applied to major sections.
- Hover/focus states for buttons, links, cards, and language switch.
- Export-ready project thumbnails or clear placeholders.

## Content To Prepare

- Name, role, location, email.
- Vietnamese and English hero copy.
- 3-6 projects with title, short summary, detailed description, tech stack, demo link, repo link, and image.
- Skill groups.
- Experience timeline.
- CV PDF.

## Design System Recommendation

Keep the design system compact:

- Tokens: color, typography, spacing, radius, shadow, z-index.
- UI primitives: Button, Badge, Card, Container, SectionHeading.
- Layout components: Header, Footer.
- Page sections: Hero, Capabilities, FeaturedProjects, LearningPlayground, CraftProcess, Experience, Contact.

Avoid designing every section as a unique one-off. Reusable rules make the site easier to animate and maintain.
